```bash
# Check OpenMPI directory structure
ls -la /usr/lib64/openmpi/
ls -la /usr/lib64/openmpi/bin/
```

Based on your include path, OpenMPI is likely installed in:
- **Include directory**: `/usr/include/openmpi-x86_64`
- **Library directory**: `/usr/lib64/openmpi/lib` or `/usr/lib64/openmpi`
- **Binary directory**: `/usr/lib64/openmpi/bin`

## 2. Set up environment variables

Add these to your `~/.bashrc`:

```bash
# OpenMPI configuration
export MPI_HOME=/usr/lib64/openmpi
export PATH=$MPI_HOME/bin:$PATH
export LD_LIBRARY_PATH=$MPI_HOME/lib:$LD_LIBRARY_PATH
export CPATH=/usr/include/openmpi-x86_64:$CPATH
export LIBRARY_PATH=$MPI_HOME/lib:$LIBRARY_PATH

# For compiling with mpicc/mpic++
export OMPI_CC=gcc
export OMPI_CXX=g++
```

Then apply the changes:
```bash
source ~/.bashrc
```

## 3. Verify the setup

```bash
# Check if mpirun is now accessible
which mpirun
mpirun --version

# Check compiler wrappers
which mpicc
mpicc --version
```

## 4. Configure Eclipse CDT with the correct paths

Now let's configure Eclipse with the exact paths we found:

### Project Properties Setup:

0. **Right-click your project** → **Properties**

1. **C/C++ Build** → **Settings**

2. **Under "GCC C++ Compiler"** → **Includes**:
   - Add: `/usr/include/openmpi-x86_64`

3. **Under "GCC C Compiler"** → **Includes**:
   - Add: `/usr/include/openmpi-x86_64`

4. **Under "GCC C++ Linker"** → **Libraries**:
   - Library search path (`-L`):
     - Add: `/usr/lib64/openmpi/lib`
   - Libraries (`-l`):
     - Add: `mpi`
     - Add: `mpi_cxx` (for C++ projects)

5. **Under "GCC C Linker"** → **Libraries**:
   - Library search path:
     - Add: `/usr/lib64/openmpi/lib`
   - Libraries:
     - Add: `mpi`

### Alternative: Use MPI Compiler Wrappers

Instead of manual path configuration, you can use the MPI compiler wrappers:

0. **Project** → **Properties** → **C/C++ Build** → **Settings**

1. **Under "GCC C++ Compiler"**:
   - Command: `mpic++`
   - All options: `${FLAGS}` (remove any -I flags)

2. **Under "GCC C Compiler"**:
   - Command: `mpicc`

3. **Under "GCC C++ Linker"**:
   - Command: `mpic++`

4. **Under "GCC C Linker"**:
   - Command: `mpicc`

## 5. Create a Run Configuration

0. **Run** → **Run Configurations**

1. **Right-click "C/C++ Application"** → **New**

2. **Name**: "MPI Run"

3. **Main tab**:
   - Project: Your project name
   - C/C++ Application: Browse to your executable (e.g., `Debug/project_name`)

4. **Arguments tab**:
   - Program arguments: `-np 4`

5. **Environment tab**:
   - Add variable:
     - Name: `LD_LIBRARY_PATH`
     - Value: `/usr/lib64/openmpi/lib`
   - Add variable (if needed):
     - Name: `PATH`
     - Value: `/usr/lib64/openmpi/bin:${PATH}`

6. **Apply** and **Run**

7. **Use ldconfig to Add System-Wide Path**

**Create a conf file for OpenMPI:**

- echo "/usr/lib64/openmpi/lib" | sudo tee /etc/ld.so.conf.d/openmpi.conf

**Update the library cache:**

- sudo ldconfig

**Verify it's now found:**

- ldconfig -p | grep libmpi.so.40

**Run your program:**

~$ ./mpi_test

8. **Why This Happens:**
   
- OpenMPI 4.0 used libmpi.so.40;
- OpenMPI 5.0 uses libmpi.so.50;
- Your program was compiled/linked expecting version 4.0;
- Recompiling with mpicc or correct paths links against the correct version 5.0 libraries;
- The mpicc wrapper automatically adds the correct include paths and library paths for your installed OpenMPI version.

9. **Test with a simple MPI program**

Create a test file `mpi_test.c`:

```c
#include <mpi.h>
#include <iostream>

int main(int argc, char** argv) {
    MPI_Init(&argc, &argv);
    
    int rank, size;
    MPI_Comm_rank(MPI_COMM_WORLD, &rank);
    MPI_Comm_size(MPI_COMM_WORLD, &size);
    
    char processor_name[MPI_MAX_PROCESSOR_NAME];
    int name_len;
    MPI_Get_processor_name(processor_name, &name_len);
    
    std::cout << "Hello from processor " << processor_name 
              << ", rank " << rank << " of " << size << std::endl;
    
    MPI_Finalize();
    return 0;
}
```

## 7. Compile and run from terminal (for testing)

```bash
# Compile with mpic++
mpic++ -o mpi_test mpi_test.c

# Run with 4 processes
mpirun -np 4 ./mpi_test
```
